QUICK FIX SUMMARY: Google OAuth Error 401 invalid_client
=========================================================

PROBLEM:
- Error: 401: invalid_client - "The OAuth client was not found"
- Site: https://wealthsync-ig8q.onrender.com

ROOT CAUSE:
- Authlib missing explicit Google OAuth endpoints
- Only using server_metadata_url for discovery (unreliable in some cases)

EXACT CODE CHANGE MADE:
=======================

File: app.py (lines 57-62)

BEFORE:
google = oauth.register(
    name="google",
    client_id=os.environ.get("GOOGLE_CLIENT_ID"),
    client_secret=os.environ.get("GOOGLE_CLIENT_SECRET"),
    server_metadata_url="https://accounts.google.com/.well-known/openid-configuration",
    client_kwargs={"scope": "openid email profile"},
)

AFTER:
google = oauth.register(
    name="google",
    client_id=os.environ.get("GOOGLE_CLIENT_ID"),
    client_secret=os.environ.get("GOOGLE_CLIENT_SECRET"),
    server_metadata_url="https://accounts.google.com/.well-known/openid-configuration",
    authorize_url="https://accounts.google.com/o/oauth2/v2/auth",
    token_url="https://oauth2.googleapis.com/token",
    client_kwargs={"scope": "openid email profile"},
)

WHAT WAS ADDED:
- authorize_url="https://accounts.google.com/o/oauth2/v2/auth"
- token_url="https://oauth2.googleapis.com/token"

WHY THIS FIXES IT:
- Explicit endpoints ensure Authlib knows where to send requests
- Removes dependency on slow/unreliable metadata discovery
- Guarantees compatibility with Google's OAuth 2.0 implementation

ADDITIONAL ENDPOINT ADDED:
===========================

GET /debug/oauth

Returns JSON with:
- client_id_loaded: true/false
- client_secret_loaded: true/false  
- redirect_uri_template: https://...
- app_url_scheme: https
- google_client_config: (starts with values)

Use this to verify credentials are loaded in Render.
